# Session Summary - 2026-04-23

## Context

- The user requested repository-local rules for future work.
- The rules require daily session summaries in `docs/session-summaries/`, consideration of those summaries before future problem solving, and no Markdown files outside `docs/` except the main `README.md`.
- The user later clarified that `AGENTS.md` is also allowed at the repository root.
- Existing repository documentation was reviewed, including `docs/first_discussion_of_idea.md`.
- The user asked to read `assignment.md` and create rules for this agent and other agents accordingly.
- `assignment.md` was found at `docs/assignment.md`.
- The user asked to make all rules readable for other agents as well, not just Codex internally.
- The user requested a rule to always use Conventional Commits.
- The user requested a rule to keep updating `README.md` during development, but only with high-level information.
- The user requested initialization of React Native, Expo, Convex, and Clerk, with separate commits for each setup step where practical.

## Decisions

- Daily summaries will be stored as one Markdown file per calendar day in `docs/session-summaries/`.
- Future work should review existing session summaries before making decisions or attempting fixes.
- The root `README.md` will hold important development information and repo-local documentation rules.
- `AGENTS.md` is now an allowed root-level Markdown exception and should hold persistent agent instructions.
- Assignment-derived constraints should live in `AGENTS.md` so future agents see them before working.
- Important assignment notes should also be summarized in `README.md` for human visibility.
- Future implementation work should preserve information needed for the Learning Journal and final presentation.
- `AGENTS.md` should be written as a shared operating contract for all agents, automation, and human collaborators.
- Commit messages must use Conventional Commits.
- `README.md` should be updated during development with high-level information only; detailed logs should stay in session summaries.
- React Native and Expo were initialized together using Expo's standard app scaffold because Expo projects are React Native applications.
- Convex was initialized locally without binding to a cloud deployment because `npx convex dev` required interactive login.
- Clerk was initialized as an Expo auth provider without forcing sign-in screens yet, so app screens can continue rendering while auth-specific UX is designed.

## Problems

- The initial Markdown placement rule did not mention `AGENTS.md`, creating a conflict with the standard Codex repo-instruction file.
- The user clarified that `AGENTS.md` can be created.
- The assignment requires transparency about AI usage through Learning Journal prompt documentation.
- Evaluation criteria include code documentation, conventions, package structure, file size, commit granularity, and a glossary, so normal implementation work must account for those constraints.
- The previous wording did not explicitly state that the rules apply to all agents and collaborators.
- Commit granularity was documented, but commit message format was not.
- Without a scope limit, the README-update rule could encourage noisy documentation churn.
- `create-expo-app` did not accept the `--no-agents-md` flag in the installed CLI version, so the scaffold had to be created in a temporary directory and copied into the repo while excluding generated agent/readme files.
- `npx convex dev --once --configure new --dev-deployment local` attempted interactive Convex login and could not run in the non-interactive terminal.
- The generated Expo scaffold needed a CSS module declaration for `npx tsc --noEmit` to pass.
- `npm audit` reported moderate vulnerabilities in the generated dependency tree; no automatic force fix was applied because that could introduce breaking dependency changes.

- Created `docs/session-summaries/2026-04-23.md` to begin the daily summary practice.
- Recorded the Markdown placement rule and the requirement to consult summaries before future work.
- Created `AGENTS.md` at the repository root with persistent instructions for session summaries, summary review, and Markdown file placement.
- Updated `README.md` so its documented exceptions match the clarified rule.
- Read `docs/assignment.md`.
- Added assignment constraints to `AGENTS.md`, including submission deadline, GitLab expectations, Learning Journal support, presentation topics, and engineering quality rules.
- Updated `README.md` with a concise assignment note for important visible development information.
- Reworded `AGENTS.md` so it explicitly applies to every AI agent, automation, and human collaborator working in this repository.
- Added a dedicated Conventional Commits rule to `AGENTS.md`.
- Added a concise Conventional Commits requirement to `README.md`.
- Added a rule to update `README.md` during development only for high-level information and keep detailed history in session summaries.
- Committed documentation rules with `docs: add repository operating rules`.
- Initialized the Expo React Native app with `create-expo-app` SDK 55 and committed it with `feat(app): initialize expo react native app`.
- Installed and wired Convex with a starter schema, `EXPO_PUBLIC_CONVEX_URL`, and a root provider, then committed it with `feat(convex): initialize backend integration`.
