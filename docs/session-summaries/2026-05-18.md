## Convex Dev Environment Support

- User asked how to initialize the Convex dev environment.
- Reviewed local Convex guidance and current repo setup.
- Clarified that Convex is already installed in the repository; initialization means running the Convex CLI login/configuration flow, ensuring `.env.local` contains `CONVEX_DEPLOYMENT` and `EXPO_PUBLIC_CONVEX_URL`, and keeping `npx convex dev` running during app development.
- User clarified that a freshly cloned colleague environment hit Convex schema validation failure.
- Diagnosed likely cause as a mismatch between the latest committed schema and data already present in the colleague's selected Convex dev deployment, especially older documents missing required fields like `clubs.slug`, `events.clubId`, `events.title`, or `events.startsAt`.
- User provided the concrete error: a `clubs` document contains `addressLine`, `city`, `postalCode`, and `source`, while the colleague's active validator only allows `description`, `lastScrapedAt`, `latitude`, `longitude`, `name`, `slug`, and `websiteUrl`.
- This confirms the colleague is running an older/local schema version against newer data; current `main` includes those club fields as optional in `convex/schema.ts`.
- User asked whether Expo prebuild must be run before starting the Android emulator.
- Checked the repo scripts and local filesystem; `android/` exists locally on this machine, but is ignored by Git and absent from fresh clones.
- Normal startup after native generation should use `npm run android:dev`, which starts/reuses the emulator and invokes `npx expo run:android`.
- Manual `npx expo prebuild --platform android` is required once on fresh clones and later only when the native Android project is missing or native Expo config/plugins need to be regenerated intentionally.
- User clarified a colleague's fresh clone failed with `Android directory not found`.
- Checked Git tracking and `.gitignore`; `android/` is ignored and not committed, so fresh clones do need a one-time `npx expo prebuild --platform android` before using `npm run android:dev`.
- Updated `README.md` with the fresh-clone Android prebuild step and changed `scripts/android-sdk.js` to report the exact prebuild command when `android/` is missing.
- User asked for the proper way to install `adb`.
- Checked official Android documentation; `adb` is included in Android SDK Platform-Tools, and Android developers should install/update it through Android Studio's SDK Manager or `sdkmanager`.
- User shared a colleague screenshot where `npm run android:dev` starts `Medium_Phone` but times out waiting for the emulator to appear in ADB.
- Diagnosed this as an emulator/ADB registration problem: the AVD launch command returned, but `adb devices` never reported an online `emulator-*` device within the script timeout.
- User asked whether the setup works on a MacBook.
- Determined that the Expo/React Native/Convex app stack can work on macOS, but the repo's current Android helper scripts are still Windows-oriented in several places (`emulator.exe`, `USERPROFILE`, Windows SDK fallback), so Mac users should either use standard Expo/Android commands or the scripts need a portability patch.
