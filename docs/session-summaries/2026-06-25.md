## Map Drawer Drag-Start Flicker Fix

- Time: 18:42 CEST, approximate duration 10 minutes.
- User reported that the discover map drawer flickers for 1-2 frames to another position when dragging starts.
- Root cause: the bottom sheet stored `currentTranslateY` as the spring target immediately when an animation started, so a new drag could use a target position rather than the current on-screen animated position as its baseline.
- Fixed `DiscoverBottomSheet` so the stored translate value tracks the animated value through an `Animated.Value` listener, only commits the final snap position after the spring finishes, and ignores move events until `stopAnimation` has initialized the real drag baseline.
- Validation:

  - `npm run lint` passed,
