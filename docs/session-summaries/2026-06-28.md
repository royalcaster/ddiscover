## Profile Favorites Loading Flicker Fix

- Time: 14:04 CEST, approximate duration 15 minutes.
- User reported that the profile Favorites section flickered between `Favoriten werden geladen...` and the empty favorites state.
- Root cause: `useFavorites` depended directly on Clerk's `getToken` function through the `refresh` callback. If Clerk returned a new `getToken` function identity across renders, the effect restarted the authenticated Convex HTTP query repeatedly, causing loading/empty-state flicker.
- Updated `src/hooks/use-favorites.ts`:
  - stores the latest Clerk `getToken` function in a ref so token retrieval stays current without changing the refresh callback identity,
  - creates fresh empty favorite id sets instead of reusing one shared object,
  - tracks whether the first signed-in favorites load has completed,
  - keeps `isLoading` true for signed-in users until that first load settles, avoiding a transient empty-state render.
- Verified that the installed Convex HTTP client supports constructor `auth` tokens for short-lived clients, so the token delivery path remains valid.

## Discover Marker Tap Responsiveness

- Time: 14:03 CEST, with 15:27 CEST follow-up; approximate duration 30 minutes.
- User reported that tapping map markers felt delayed and asked for the markers to react faster.
- Reviewed the MapLibre marker path and recent map performance notes before changing the implementation.
- Root cause addressed:
  - selected marker state was encoded into the full marker GeoJSON source, so each selection required updating the full marker collection,
  - Android MapLibre source presses still arrive through the native map click path, which can feel slower than a direct touch response.
- Updated `src/components/discover-map.tsx`:
  - split the selected marker into its own one-feature GeoJSON source/layers so visual selection updates only a tiny source,
  - added a React Native touch fast path on the map wrapper that treats low-movement taps as marker taps by projecting known marker coordinates into the current viewport and selecting the nearest marker immediately,
  - increased the MapLibre source hitbox from 32x32 to 48x48 pixels as the exact native fallback.
- Follow-up after the user asked for even more responsiveness:
  - marker selection now happens on touch start when the finger lands inside a marker radius, instead of waiting for touch end,
  - delayed MapLibre source press callbacks are deduplicated through a selected-marker ref,
  - the fast hit radius and native fallback hitbox were widened,
  - Discover screen callbacks passed to the map were stabilized with `useCallback`.
